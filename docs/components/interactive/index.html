InteractiveManager

implements IDisposable

基于 Raycaster 的集中式事件交互系统，忠实改编自 react-three-fiber 的 createEvents。开发者只需注册 Object3D + 事件回调，无需关心 Raycaster、NDC 坐标转换、hover diffing 等底层原理。支持 click / doubleclick / pointerdown / pointerup / pointermove / pointerover / pointerout / pointerenter / pointerleave / wheel / contextmenu / pointermissed 全套事件，内置扁平列表传播、点击 vs 拖拽判别、指针捕获与相机控制器自动抑制。

R3F 事件模型

扁平传播（非 DOM 冒泡）：每次射线检测产生按 distance 排序的 Intersection[]，每个原始命中会向上展开到所有注册祖先。事件按此扁平列表顺序依次派发，stopPropagation() 中断后续迭代。

Hover Diffing：通过 composite-id（eventObject.uuid/faceIndex/instanceId）追踪悬停状态。进入新对象时 over+enter 同时触发；离开时 out+leave 同时触发（由 cancelPointer 执行）。

Click 验证：onClick 仅在 pointerDown 时命中的对象（initialHits）上触发。pointerMissed 在未命中的注册对象上触发。

Methods

.add(object: Object3D, handlers: EventHandlers): void

注册一个 Object3D 及其事件回调。若已注册则更新回调。

取消注册，若当前悬停则触发 out/leave 后移除。

重叠区域交互行为

当两个独立的注册对象（非父子关系）在空间中部分重叠时：

  • A区（仅小球）：射线只命中小球 → 只触发小球的事件
  • B区（重叠处）：射线同时命中两个对象 → 两个对象都在 intersections 列表中，事件按 distance 排序依次派发
  • C区（仅大球）：射线只命中大球 → 只触发大球的事件

进入/离开重叠区时，hover diffing 基于 composite-id 独立追踪每个对象，互不影响。

在重叠区，stopPropagation() 可以阻止距离更远的对象接收事件。

示例

import { InteractiveManager } from '@cyc/3d-components/interactive';

const manager = new InteractiveManager({
  camera,
  domElement: renderer.domElement,
  scene,
  controls: orbitControls,
});

manager.add(myMesh, {
  onClick: (e) => console.log('clicked!', e.eventObject),
  onPointerOver: (e) => { e.eventObject.material.emissive.setHex(0x333333); },
  onPointerOut: (e) => { e.eventObject.material.emissive.setHex(0x000000); },
  onPointerMissed: (e) => console.log('missed!'),
});

// Pointer capture for drag
manager.add(draggableMesh, {
  onPointerDown: (e) => {
    e.target.setPointerCapture(e.nativeEvent.pointerId);
  },
  onPointerMove: (e) => { /* drag logic */ },
  onPointerUp: (e) => {
    e.target.releasePointerCapture(e.nativeEvent.pointerId);
  },
});

manager.dispose();
