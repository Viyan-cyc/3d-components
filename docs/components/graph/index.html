Graph3D

extends BaseGroup（THREE.Group） implements IUpdatable · IDisposable

基于图数据结构（节点 Node + 边 Edge）的 3D 可视化组件，设计理念参考 AntV G6，渲染引擎基于 Three.js。本页 demo 把 4 个布局算法（circular / force / hex / grid）各做成一个 Tab，每个 Tab 配该布局专属的实时参数滑块 —— 切 Tab 即 graph.setLayout(preset) 声明式重排（被记忆，setData 后自动重应用）。force 支持 Barnes-Hut 八叉树加速（barnesHut:true，斥力 O(n²)→O(n log n)，大图提速）。

上方 demo 默认渲染一组示例图（约 12 节点）。左侧面板把 4 个布局算法各做一个 Tab —— 环形 / 力导向 / 六边形 / 网格，每个 Tab 配该布局专属的实时参数滑块（半径/斥力/层数/列数 等）。切 Tab 经 graph.setLayout(preset) 声明式切换并被记忆，节点经 gsap 过渡平滑飞抵目标坐标（同一批节点在布局间「飞」）；拖滑块即时重排（力导向因昂贵在释放时重算）。右侧面板是全局控件：点「重新生成」会自动重应用当前 Tab 布局（无需手动 apply，Step 5 核心）；切换「边形态」在直线/管道(带箭头)/混合间切换；力导向 Tab 可勾「Barnes-Hut」开启八叉树加速，切到大节点数（最高 2000）时面板内「计算耗时」行显示提速效果。悬停节点放大发光、边提亮；点击节点/边选中（节点常亮发光、边常亮+emissive）。切换「单选/多选」验证互斥与累加；选中节点只提亮邻接边（其余边不变）；点击空白/地面清除选中；OrbitControls 拖拽不会误触发点击。

Methods

.setData(data: GraphData): void

校验并归一化数据，释放旧元素后重新生成球体节点与直线段边。初始坐标取 data.x/y/z，未指定则在原点环形散布。

.getNodes(): Node3D[]  ·  .getNode(id): Node3D | undefined

访问节点元素实例（便于后续步骤追加交互）。

.getEdges(): Edge3D[]  ·  .getEdge(id): Edge3D | undefined

访问边实例。Edge3D 支持 type:'line'|'path'、arrow，getMaterial() 返回该边独立 clone 的材质（可直接改色/透明度做反馈）。

.getData(): GraphData | null  ·  .getIndex(): GraphIndex | null

访问规范化数据/索引（邻接表、incidentEdges）。

.applyLayout(layout, config?, options?): void  ·  .applyPositions(positions, options?): void

应用一次布局算法刷新全部节点坐标与边端点。applyLayout 接收布局函数（如 Layouts.circular/Layouts.force）+ 配置；applyPositions 跳过计算直接置位（外部坐标）。options.animate 启用 gsap 过渡动画（默认关），duration、onComplete 可选。力导向自动注入当前图边 —— graph.applyLayout(Layouts.force) 即可。一次性应用，不持久化（持久化 setLayout 留 Step 5）。

.setLayout(preset: LayoutPreset | null, options?): void  ·  .getLayout(): LayoutPreset | null

声明式布局 API（Step 5）：setLayout 切换并记忆预设，后续 setData 自动重应用（无需手动 applyLayout）；传 null 清除（退回占位散布）。getLayout 读取当前预设。与 applyLayout 区别：applyLayout 一次性不记忆；setLayout 持久化、重建自动编排。

.setNodeGeometry(factory: ((size: number) => BufferGeometry) | null): void

运行时切换节点几何工厂，就地重建所有现存节点几何（坐标/材质不变）；传 null 回退球体。工厂被记忆，setData 重建后仍套用（「重新生成」后自定义形状不丢）。本页六边形 Tab 即用它把节点换成六边形瓦片以凸显蜂巢拼合。

.update(delta: number): void  ·  .dispose(): void

update 每帧由渲染循环调用，转发给子元素；dispose 释放全部节点/边资源（并终止进行中的布局过渡动画）。

布局 · Layouts（Step 3-5）

Layouts.circular / Layouts.force / Layouts.hex / Layouts.grid  —  (nodes, config?) → NodePos3D[]

纯函数、零 Three.js 运行时依赖、可独立调用（不依赖 Graph3D）。统一签名 (nodes, config) => NodePos3D[]，输出含完整 x/y/z。命名空间风格 Layouts.xxx(...) 与命名导入 import { hex } from ... 等价。声明式使用见 LayoutPreset（Step 5），经 setLayout/Graph3DOptions.layout 接入。

布局关键配置说明
circularradius(3) · startAngle/endAngle · plane('xz') · rings(1)+radiusStep · groupBy · depthOffset/layerSpacing2D 圆周按 plane 映射三维。groupBy 命中时每组一个深度层；否则 rings>1 走同心多环；其余单圈单层
forcedimensions(3) · iterations(300) · linkDistance(1)/linkStrength(0.3) · chargeStrength(30, 正=斥) · centerStrength(0.02) · center · velocityDecay(0.6) · edges · barnesHut(false)Step5 · theta(0.9)Step5d3-force 思路三维力导向。dimensions:2 走 plane 映射；edges 经配置传入（applyLayout 自动注入）。恒返回有限坐标。barnesHut:true（仅 3D）启用八叉树近似 O(n log n) 加速大图；theta 越小越精确越慢
hexradius(1, 外接圆) · orientation('flat'|'pointy', 默认 'flat') · layers(1) · groupBy · plane('xz') · depthOffset/layerSpacing六边形蜂巢：轴向坐标 (q,r) 从中心逐环螺旋铺开（移植 RedBlobGames），按 orientation 换算后经 plane 映射三维。layers>1 按 index 轮询多层堆叠；groupBy 命中时每组一层。本质 2D 布局
gridcols/rows/levels(1) · spacingX/spacingY/spacingZ(均 1)行/列/层三维网格：col→x、row→z、level→y，居中于原点。缺省自动推算 cols/rows 铺满。本质 3D 布局（plane/depthOffset/layerSpacing 为 no-op）

声明式布局 · LayoutPreset（Step 5）

type LayoutPreset = { type: 'circular'|'force'|'hex'|'grid'; config?: ... }

判别联合：type 决定 config 具体类型，调用点即获类型安全。经 Graph3DOptions.layout（构造时）或 setLayout（运行时）接入 —— 布局被记忆，setData 后自动重应用。与命令式 applyLayout（一次性、不记忆）互补：声明式适合「布局随数据变化自动重排」场景，命令式适合「一次性手动驱动」场景。

交互 · PickController

new PickController({ domElement, graph, camera, onHover?, onSelect?, highlightOnHover?, highlightOnSelect?, neighborHighlight? })

监听 canvas pointer 事件，Raycaster 拾取 Graph3D 子树（节点优先于边回查 userData.nodeId/edgeId）。节点与边均可被选中：节点 hover 放大+发光、select 常亮；边 hover/select 提亮为强调色（path 形态加 emissive）。选中模式 single（互斥）/ multiple（纯累加 toggle）运行时可切换。选中节点时只提亮其邻接边（其余边不变）。点击空白/地面自动清空选中；另暴露 clearSelection()、getSelectedNodes()、getSelectedEdges()。分发 GraphEvent（hover/unhover/click/select/unselect）给 onHover/onSelect 回调。需在销毁时调用 pick.dispose()。

数据类型

interface GraphData {
  nodes: NodeData[];   // { id, x?, y?, z?, size?, type?, data? }
  edges: EdgeData[];   // { source, target, id?, type?, data? }
}

// 所有布局算法的统一输出 —— 含完整 x/y/z
interface NodePos3D { id: string | number; x: number; y: number; z: number; }

示例

import { Graph3D, PickController, Layouts } from '@cyc/3d-components/graph';

const graph = new Graph3D({ nodeSize: 0.4, edgeType: 'path', edgeArrow: true });
graph.setData({
  nodes: [{ id: 'n1' }, { id: 'n2' }, { id: 'n3' }],
  edges: [
    { source: 'n1', target: 'n2', type: 'path' },
    { source: 'n2', target: 'n3', type: 'line' },
  ],
});
scene.add(graph);

// 声明式布局（Step 5）：构造时指定 → setData 自动应用并记忆
const graph = new Graph3D({
  nodeSize: 0.4, edgeType: 'path', edgeArrow: true,
  layout: { type: 'force', config: { iterations: 300, barnesHut: true } }, // 大图加速
});
graph.setData({
  nodes: [{ id: 'n1' }, { id: 'n2' }, { id: 'n3' }],
  edges: [
    { source: 'n1', target: 'n2', type: 'path' },
    { source: 'n2', target: 'n3', type: 'line' },
  ],
}); // ← 自动跑力导向，无需再 applyLayout
scene.add(graph);

// 运行时切换布局（被记忆，后续 setData 自动重应用）
graph.setLayout({ type: 'hex', config: { radius: 1.3, layers: 3, layerSpacing: 2.4 } });
graph.setLayout({ type: 'grid', config: { levels: 3, spacingY: 2.4 } });

// 命令式一次性应用（不记忆，适合手动驱动）
graph.applyLayout(Layouts.circular, { radius: 4, plane: 'xz' });

// 布局函数也可独立调用（纯函数、零 Three.js 依赖）：
// const positions = Layouts.force(nodes, { edges, dimensions: 3, barnesHut: true });
// const honeycomb = Layouts.hex(nodes, { radius: 1.3, orientation: 'pointy' });
// const grid3d    = Layouts.grid(nodes, { cols: 4, levels: 2 });

// 交互拾取 + 内置反馈 + 事件回调
const pick = new PickController({
  domElement: renderer.domElement, graph, camera,
  onHover: (e) => console.log('hover', e.kind, e.id),
  onSelect: (e) => console.log(e.type, e.kind, e.id),
});

// 渲染循环中：
graph.update(delta); pick.update(delta);
// 销毁：pick.dispose(); graph.dispose();
